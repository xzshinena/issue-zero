  datasets exist for those tasks. Currently all 3 remain on sklearn LR.

## Failure Modes

| Scenario | Current behavior | Fix |
|----------|-----------------|-----|
| No issues in DB | BM25 index is None, vector search returns [] | Returns empty pack — correct |
| `OPENAI_API_KEY` missing with provider=openai | ValueError on first embed call | Error bubbles to 500 — add health check |
| GitHub rate limit during sync | `requests.get()` raises — unhandled in tasks.py | Celery retry / logger.exception (Issue 4) |
| `models/*.joblib` missing | `_try_load_trained()` returns None, heuristic fallback | Correct — works without trained models |
| BM25 cold start under load | First request takes 2-10s (corpus load) | Add startup warm-up in `lifespan()` |
| Large issue body (>512 tokens) | Chunked — but vector search only uses chunk_id IS NULL | Chunk embeddings are stored but not searched |
| Malformed GitHub issue JSON in training data | Crash with no message (pre-fix) | Fixed by Issue 8 |
